# Contributing to ARIA Investor Site

Thank you for your interest in the ARIA platform. This repository contains investor presentation materials for the ARIA CPU Platform and is **not accepting external code contributions**.

## What This Repository Contains

This is a static GitHub Pages site presenting:
- Executive overview and technical architecture
- Market analysis and revenue projections
- Partnership opportunities
- Contact information for investor inquiries

## Accepted Contributions

We accept issues and pull requests **only** for the following:

### Documentation Corrections
- Spelling or grammar errors
- Broken links
- Formatting issues
- Factual inaccuracies in publicly available information

### Accessibility Improvements
- Alt text for images
- Screen reader compatibility
- Color contrast issues

## Not Accepted

The following are **not accepted** in this repository:

- Feature requests for the ARIA platform itself
- Code contributions or architectural suggestions
- Security vulnerability reports (see [SECURITY.md](SECURITY.md))
- Requests for proprietary technical details
- Marketing or promotional content

## How to Submit an Issue

1. Check existing issues to avoid duplicates
2. Use the appropriate issue template:
   - **Bug Report** for site errors or broken functionality
   - **Feature Request** for documentation improvements
3. Provide clear, specific information
4. Include screenshots if applicable
5. Note the URL where the issue occurs

## Pull Request Process

If submitting a documentation fix:

1. Fork the repository
2. Make your changes in a feature branch
3. Ensure all content is accurate and professional
4. Verify no proprietary information is included
5. Submit a pull request using the provided template
6. Complete all checklist items in the PR template

Pull requests will be reviewed by the project maintainer. Expect a response within 5 business days.
